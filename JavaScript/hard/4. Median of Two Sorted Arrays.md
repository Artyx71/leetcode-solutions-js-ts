# 4. Median of Two Sorted Arrays

## Алгоритм (RU)

**Подход:** Слияние двух отсортированных массивов (виртуальное) — O(m+n).

Не создаём новый массив. Проходим по обоим массивам одновременно, как при merge sort, отслеживая только нужные позиции медианы.

- **Сложность по времени:** O(m+n)
- **Сложность по памяти:** O(1)

**Шаги:**
1. Вычисляем индексы медианы: `med1 = floor((total-1)/2)`, `med2 = floor(total/2)`
   - Чётная длина: `med1 ≠ med2`, нечётная: `med1 == med2`
2. Итерируемся по виртуально слитому массиву до позиции `med2`
3. На каждом шаге берём минимум из текущих голов `arr1[j1]` и `arr2[j2]`
4. Сохраняем элементы на позициях `med1` и `med2`
5. Возвращаем `(medians[0] + medians[1]) / 2`
   - Нечётная длина: оба элемента одинаковые → среднее = сам элемент

---

## Algorithm (EN)

**Approach:** Virtual merge of two sorted arrays — O(m+n).

No new array created. Traverse both arrays simultaneously (like merge sort), tracking only the required median positions.

- **Time complexity:** O(m+n)
- **Space complexity:** O(1)

**Steps:**
1. Compute median indices: `med1 = floor((total-1)/2)`, `med2 = floor(total/2)`
   - Even total: `med1 ≠ med2`; odd total: `med1 == med2`
2. Iterate through the virtually merged array up to position `med2`
3. At each step take the minimum of current heads `arr1[j1]` and `arr2[j2]`
4. Store elements at positions `med1` and `med2`
5. Return `(medians[0] + medians[1]) / 2`
   - Odd total: both elements are equal → average = the element itself
