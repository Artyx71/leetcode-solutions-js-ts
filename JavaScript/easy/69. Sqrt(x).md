# 69. Sqrt(x)

## Алгоритм (RU)

**Подход:** Метод Ньютона (Newton's Method / Babylonian method).

Итеративно уточняем приближение корня по формуле `x = (x + n/x) / 2` до сходимости. Затем берём целую часть и проверяем граничный случай.

- **Сложность по времени:** O(log n) — квадратичная сходимость
- **Сложность по памяти:** O(1)

**Шаги:**
1. Граничные случаи: `n < 0` → `NaN`, `n === 0` → `0`
2. Начальное приближение `x = n`
3. Итерируем: `x = (x + n/x) / 2` пока `|x - prev| > 1e-7`
4. `floorResult = trunc(x)` — отбрасываем дробную часть
5. Проверяем: если `(floorResult + 1)² === n` — возвращаем `floorResult + 1`
6. Иначе возвращаем `floorResult`

> Шаг 5 нужен для точных квадратов (например, n=4, n=9), где погрешность float может дать `floor` на 1 меньше правильного ответа.

---

## Algorithm (EN)

**Approach:** Newton's Method (Babylonian method).

Iteratively refine the root approximation using `x = (x + n/x) / 2` until convergence. Then truncate and handle the edge case.

- **Time complexity:** O(log n) — quadratic convergence
- **Space complexity:** O(1)

**Steps:**
1. Edge cases: `n < 0` → `NaN`, `n === 0` → `0`
2. Initial guess `x = n`
3. Iterate: `x = (x + n/x) / 2` while `|x - prev| > 1e-7`
4. `floorResult = trunc(x)` — drop the fractional part
5. Check: if `(floorResult + 1)² === n` — return `floorResult + 1`
6. Otherwise return `floorResult`

> Step 5 handles perfect squares (e.g. n=4, n=9) where floating-point drift may cause `floor` to be 1 less than the correct answer.
