# 5. Longest Palindromic Substring

## Алгоритм (RU)

**Подход:** Алгоритм Манакера (Manacher's Algorithm) — O(n).

Превращает задачу поиска палиндромов в линейную, избегая повторного сравнения символов. Вставляет разделители `#` между символами, чтобы унифицировать чётные и нечётные палиндромы.

- **Сложность по времени:** O(n)
- **Сложность по памяти:** O(n) — массив `P` и трансформированная строка

**Шаги:**
1. Трансформируем строку: `"abc"` → `"^#a#b#c#$"` (sentinel-символы `^` и `$` исключают проверку границ)
2. `P[i]` — радиус палиндрома с центром в `i`
3. Поддерживаем `center` и `right` — центр и правая граница самого правого известного палиндрома
4. Для каждой позиции `i`:
   - Если `i < right`: `P[i] = min(right - i, P[mirror])` — используем симметрию
   - Расширяем `P[i]` пока символы совпадают
   - Если новый палиндром выходит за `right` — обновляем `center` и `right`
5. Находим максимальный `P[i]`, вычисляем начало в оригинальной строке: `start = (centerIndex - maxLen) / 2`

## Algorithm (EN)

**Approach:** Manacher's Algorithm — O(n).

Solves palindrome search in linear time by avoiding redundant character comparisons. Inserts `#` separators between characters to unify even- and odd-length palindromes.

- **Time complexity:** O(n)
- **Space complexity:** O(n) — for array `P` and the transformed string

**Steps:**
1. Transform string: `"abc"` → `"^#a#b#c#$"` (sentinel characters `^` and `$` eliminate boundary checks)
2. `P[i]` = palindrome radius centered at `i`
3. Maintain `center` and `right` — center and right boundary of the rightmost known palindrome
4. For each position `i`:
   - If `i < right`: `P[i] = min(right - i, P[mirror])` — exploit symmetry
   - Expand `P[i]` while characters match
   - If new palindrome exceeds `right` — update `center` and `right`
5. Find max `P[i]`, compute start in original string: `start = (centerIndex - maxLen) / 2`
