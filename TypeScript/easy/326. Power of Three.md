# 326. Power of Three

## Алгоритм (RU)

**Подход:** Логарифмическая проверка.

`n` — степень тройки, если `log₃(n)` — целое число. Вычисляем `log₃(n) = log(n) / log(3)` и проверяем, что результат близок к целому (с учётом погрешности плавающей точки).

- **Сложность по времени:** O(1)
- **Сложность по памяти:** O(1)

**Шаги:**
1. Если `n <= 0` — сразу `false` (степень тройки всегда положительна)
2. Вычисляем `logResult = log(n) / log(3)`
3. Проверяем `|logResult - round(logResult)| < 1e-10`
   - Порог `1e-10` компенсирует ошибки плавающей точки

> Альтернатива без float: `1162261467 % n === 0`, где `1162261467 = 3¹⁹` — максимальная степень тройки в `int32`.

---

## Algorithm (EN)

**Approach:** Logarithm check.

`n` is a power of three if `log₃(n)` is an integer. Compute `log₃(n) = log(n) / log(3)` and verify the result is close to a whole number (accounting for floating-point error).

- **Time complexity:** O(1)
- **Space complexity:** O(1)

**Steps:**
1. If `n <= 0` — return `false` immediately (powers of three are always positive)
2. Compute `logResult = log(n) / log(3)`
3. Check `|logResult - round(logResult)| < 1e-10`
   - Threshold `1e-10` compensates for floating-point precision errors

> Integer alternative: `1162261467 % n === 0`, where `1162261467 = 3¹⁹` — the largest power of three within `int32`.
