# 3. Longest Substring Without Repeating Characters

## Алгоритм (RU)

**Подход:** Скользящее окно (Sliding Window) с хэш-картой.

Поддерживаем окно `[start, i]` без повторяющихся символов. Map хранит последний встреченный индекс каждого символа. Если символ уже виден — сдвигаем `start` правее последней позиции этого символа.

- **Сложность по времени:** O(n) — один проход
- **Сложность по памяти:** O(min(n, m)) — m размер алфавита

**Шаги:**
1. `seen` — Map: символ → последний индекс
2. `start = 0`, `maxLen = 0`
3. Для каждого символа `s[i]`:
   - Если символ в `seen` — `start = max(seen[s[i]] + 1, start)`
   - Обновляем `seen[s[i]] = i`
   - `maxLen = max(i - start + 1, maxLen)`
4. Возвращаем `maxLen`

> `Math.max` при обновлении `start` защищает от отката назад, если символ встречался до текущего окна.

---

## Algorithm (EN)

**Approach:** Sliding Window with a HashMap.

Maintain a window `[start, i]` with no duplicate characters. Map stores the last seen index of each character. If a character was already seen — shift `start` to the right of its last position.

- **Time complexity:** O(n) — single pass
- **Space complexity:** O(min(n, m)) — m is alphabet size

**Steps:**
1. `seen` — Map: character → last index
2. `start = 0`, `maxLen = 0`
3. For each character `s[i]`:
   - If character in `seen` — `start = max(seen[s[i]] + 1, start)`
   - Update `seen[s[i]] = i`
   - `maxLen = max(i - start + 1, maxLen)`
4. Return `maxLen`

> `Math.max` when updating `start` prevents rolling back if the character appeared before the current window.
